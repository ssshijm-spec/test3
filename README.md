# 점심 뭐 먹지: 구내식당 런! 🏃‍♂️🍚

60초 안에 끝나는 **백뷰(Back View) 픽셀아트 점심메뉴 레이싱 게임**.
위메이드플레이 직원이 구내식당까지 달려가며, 3갈래 배식대에서 점심 메뉴 카드를
고르는 **메뉴 월드컵(챔피언 브라켓)** 러너입니다. 최후에 살아남은 메뉴가
**오늘의 우승 점심 메뉴**로 발표됩니다.

- **엔진 없음**: HTML5 Canvas + 순수 JavaScript (Unity/프레임워크 미사용)
- **한 판 ≈ 52초** (인트로 3s → 레이스 35s → 완주 2s → 우승 세리머니), 60초 하드캡 이내
- **모바일/데스크톱** 모두 대응 (키보드 · 마우스 · 터치/스와이프)

---

## ▶ 실행 방법

1. **`start.bat` 더블클릭** → 로컬 서버가 뜨고 브라우저가 자동으로 열립니다.
   (Python 3 필요. 창을 닫으면 서버 종료)
2. 브라우저에서 **클릭 / 탭 / [Space]** 로 시작.

> 왜 서버가 필요한가요? `index.html` 을 파일로 직접 열면(`file://`) 브라우저 보안정책상
> 로컬 음식 사진이 canvas 에 안 그려집니다. `start.bat` 이 이 문제를 자동으로 우회합니다.
> (사진 없이 플레이스홀더만 볼 거라면 `index.html` 직접 열기도 동작합니다.)

## 🎮 조작

| 입력 | 동작 |
|------|------|
| `←` `→` 또는 `A` `D` | 좌/우 레인 이동 |
| 마우스 클릭 (좌/우 절반) | 좌/우 레인 이동 |
| 터치 탭(좌/우) · 좌우 스와이프 | 좌/우 레인 이동 |
| `R` / 탭 / 클릭 (결과 화면) | 다시 달리기 |

원하는 메뉴 카드의 레인으로 이동해 게이트를 통과하면 그 카드가 선택되고,
나머지 두 카드는 파편으로 부서집니다. 다음 라운드엔 **챔피언 1장 + 새 도전자 2장**이
다가오고, 다시 한 장을 고릅니다. 안 고르면 현재 레인의 카드가 자동 선택됩니다.

## 🍱 음식 사진 / 메뉴 커스터마이징

- **사진 추가**: `assets/menus/` 폴더에 `{slug}.png` 로 저장 → 새로고침(F5). (자세히: `assets/menus/README.txt`)
- **메뉴 수정**: `config.js` 의 `FOODS` 배열에서 이름/이모지/색상/slug 를 편집.
- 사진이 없으면 **색상 + 이모지 플레이스홀더 카드**로 자동 표시되며, 사진을 넣으면
  레이아웃 변화 없이 그대로 교체됩니다(픽셀아트 톤으로 자동 변환).

예) `config.js`
```js
{ name: '제육볶음', emoji: '🥘', color: '#E8452B', slug: 'jeyuk-bokkeum' },
//                                                        └─ assets/menus/jeyuk-bokkeum.png
```

- **메뉴 확인 시간 조절**: `config.js` 의 `raceSeconds` 를 키우면 라운드가 길어져 카드를
  더 오래 볼 수 있습니다. `roundCount` 를 줄여도 한 라운드가 길어집니다. (완주 시간 = `raceSeconds`)
- **배경**: 회사 내부 복도(천장 라인조명 · 유리문 · 폴리시 바닥)이며, 복도 끝 벽과 타이틀에
  `wemadeplay.png` 로고가 노출됩니다. 로고 이미지를 교체하면 그대로 반영됩니다.

## 🗂 파일 구조

```
index.html       진입점 (canvas 셸)
style.css        전체화면 픽셀 퍼펙트 셸 스타일
config.js        ★ 사용자 편집: 메뉴 목록 + 사진 폴더 설정
game.js          게임 엔진 (렌더/도로/러너/라운드/주스/오디오/HUD 전부)
assets/menus/    ★ 음식 사진 폴더 ({slug}.png)
serve.py         로컬 서버 (start.bat 이 호출)
start.bat        더블클릭 실행
```

## 🛠 기술 메모

- **가상 해상도 320×180** 오프스크린 버퍼에 픽셀 월드를 그린 뒤 정수 배율로 블릿 →
  어떤 화면에서도 픽셀이 뭉개지지 않음. 한글 텍스트는 디스플레이 캔버스에 별도 크리스프 렌더.
- **의사 3D 도로**: 스캔라인별 `roadHalfWidth(y)` 원근 + 공유 depth field 로 도로/카드/소품/스트라이프가 하나의 원근을 공유.
- **타임 기반 상태머신**: 마스터 시계(dt 누적)가 유일한 기준. 라운드는 경과 시간으로 스케줄되어
  입력과 무관하게 정확히 라운드 수(`roundCount`, 기본 9) 만큼 진행 후 완주 → 60초 하드캡 보장. 탭 전환 시 자동 일시정지.
- **오디오**: WebAudio 신스만 사용(효과음 + BGM), 오디오 애셋 파일 없음. 시작 클릭 제스처에서 언락.
- 게임 디자인/수치는 다중 전문가 렌즈(게임필·메카닉·아트·콘텐츠) 설계 종합 스펙 기반.
